DefectDojo

Centralna baza podatności. Przyjmuje VulnerabilityReports ze wszystkich klastrów przez REST API. Deduplikuje, śledzi trend, mapuje na produkty.

Falcosidekick (centralny)

Jeden punkt routingu alertów Falco z wszystkich klastrów. Dodaje enrichment: cluster_name, environment, cloud_provider do każdego eventu przed wysłaniem do SIEM.

Thanos + Grafana

Thanos agreguje metryki Prometheus ze wszystkich klastrów w jeden widok. Grafana: dashboardy CVE counts, compliance scores, alert rates per klaster.

Warstwa wyjściowa — raporty, SIEM, dashboardy

CVE Report (CSV / Excel)

Generowany z DefectDojo przez API lub UI. Jeden plik dla całego środowiska.

  • Filtrowanie: severity, klaster, namespace, obraz
  • Deduplikacja tych samych CVE z wielu klastrów
  • Trend: nowe vs zamknięte CVE w czasie
  • Export: CSV, Excel, PDF przez DefectDojo UI
  • Automatyczny raport cykliczny (CronJob → email)

SIEM — Runtime Events

Falcosidekick wysyła alerty Falco bezpośrednio do SIEMa z enrichmentem klastra.

  • Splunk — HEC (HTTP Event Collector)
  • Microsoft Sentinel — syslog / Logic Apps
  • QRadar — syslog listener
  • Elastic / OpenSearch — natywna integracja
  • Priority: CRITICAL → page-on-call, WARNING → ticket

Grafana — Dashboardy

Widok operacyjny dla całego środowiska w czasie rzeczywistym.

  • CVE counts per klaster / severity / namespace
  • Falco alert rate per klaster i regułę
  • Compliance score (Kubescape metrics)
  • Alertmanager → Slack / PagerDuty
  • SLA: % klastrów bez CRITICAL CVE
Uwagi projektowe
CVE pipeline: CronJob w każdym klastrze nie wymaga stałego połączenia — push jest jednostronny (klaster → DefectDojo). DefectDojo może być w osobnym VPC lub on-prem dla środowisk gdzie dane nie mogą wychodzić. Trivy Operator można skonfigurować żeby skanował tylko CRITICAL i HIGH (mniejszy ruch, mniejszy storage).
Runtime pipeline: Każdy Falcosidekick lokalny (per klaster) musi mieć skonfigurowany customfields z cluster_name i environment — bez tego w SIEMie nie wiadomo skąd przyszedł alert. Centralny Falcosidekick można zastąpić bezpośrednim wysyłaniem do SIEM z każdego sidekicka lokalnie — prostsze, ale trudniejsze do zarządzania konfiguracją przy setkach klastrów.
Skalowanie: Thanos QueryFrontend agreguje Prometheusy bez konieczności federacji — każdy klaster ma autonomiczny Prometheus, Thanos tylko odpytuje. Przy >50 klastrach warto rozważyć Grafana Mimir (managed Prometheus, lepiej skalowalny niż Thanos). Grafana Alloy (następca Fluent Bit/Agent) upraszcza konfigurację na każdym klastrze.
