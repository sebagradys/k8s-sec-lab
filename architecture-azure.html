Azure Event Hubs

Drop-in replacement dla Apache Kafka — producenci używają Kafka API bez żadnych zmian w kodzie. Microsoft zarządza brokerami, replikacją i failover. Auto-inflate: automatycznie skaluje przepustowość przy spike'ach (np. inicjalny skan Trivy wielu klastrów naraz).
Kafka-compatible API (port 9093) Schema Registry (built-in) Capture → ADLS Gen2 (auto-archiwum) Auto-inflate (auto-scale) Geo-redundancy

Azure Stream Analytics

Real-time SQL query na strumieniu eventów z Event Hubs. Zero infrastruktury do zarządzania — serverless.

  • Konsumuje sec.runtime.raw ciągle
  • SQL WHERE: priority IN ('Critical','High')
  • Allowlist: wyklucza znane operatory (Trivy, Kubescape, Arc agents) przez reference data join
  • Output: sec.alerts.critical → do SIEM consumera
  • Drugie query: aggregate alert counts per cluster per 5min → ADLS Silver
  • Latency: <1s od eventu do alertu w SIEM

Azure Databricks — Spark Streaming + Batch

Złożone przetwarzanie, enrichment i budowanie Medallion layers na ADLS Gen2.

  • CVE Enrichment Job (micro-batch 1h): deduplikacja tych samych CVE z wielu klastrów, join z CMDB (team owner per namespace), normalizacja severity → Silver
  • CVE Aggregation Job (batch 4h): counts per cluster/severity/namespace/image, compliance score (% bez CRITICAL), trend 7d/30d → Gold
  • Audit Enrichment (batch daily): kto deployował podatny obraz? join VulnReport + audit log → Silver
  • Unity Catalog: governance danych, data lineage, access control per team

Azure Data Lake Storage Gen2 + Delta Lake

Centralny storage dla wszystkich danych security. Delta Lake format (open source, natywny w Databricks) zapewnia ACID transakcje, time travel (możesz odpytać stan CVE sprzed 30 dni), schema evolution i Z-ordering dla szybkich queries w PowerBI.

Azure Synapse Analytics — Serverless SQL Pool

SQL engine który odpytuje pliki Parquet/Delta Lake na ADLS Gen2 bez ładowania danych. PowerBI łączy się przez Synapse jako SQL endpoint (DirectQuery — zawsze aktualne dane). Synapse Pipeline generuje cykliczny CSV/Excel raport z Silver layer i wysyła przez Logic App na e-mail / Teams / SharePoint.
DirectQuery endpoint dla PowerBI SQL na Parquet/Delta bez ETL Synapse Pipeline → CSV/Excel → Logic App Pay-per-query (TB scanned) Role-based access: team widzi swoje klastry

PowerBI — Dashboardy CVE + Compliance

  • DirectQuery na Gold layer przez Synapse — dane real-time
  • CVE heatmapa: klaster × severity × namespace
  • Trend 7d/30d/90d: nowe vs remediated CVE
  • Compliance score per klaster i per team
  • Top 10 najbardziej podatnych obrazów w środowisku
  • Runtime alert frequency per klaster i regułę Falco
  • Row-Level Security: team owner widzi tylko swoje
  • Scheduled snapshot → Teams / e-mail

CVE Report — CSV / Excel

  • Synapse Pipeline (cron: tygodniowo / miesięcznie)
  • Query na Silver layer: jeden plik, całe środowisko
  • Kolumny: CVE ID, severity, image, cluster, namespace, team_owner, days_open, fixed_version, cvss_score
  • Deduplikacja: ten sam CVE w 100 klastrach = 1 wiersz z listą klastrów
  • Output → ADLS Blob → Logic App → e-mail security team
  • Alternatywnie: SharePoint library (auto-sync)

SIEM — Runtime Alerts

  • Consumer topicu sec.alerts.critical (już przefiltrowanego przez Stream Analytics)

  • Elastic — Logstash Event Hubs input plugin

Kluczowe uwagi
Event Hubs Capture jako "darmowy" Bronze: Event Hubs ma wbudowaną funkcję Capture — automatycznie zapisuje wszystkie eventy do ADLS Gen2 jako Avro/Parquet bez żadnego kodu. To jest Bronze layer za darmo (płacisz tylko za storage). Databricks zaczyna pracę od Silver.
Stream Analytics vs Databricks Streaming: Stream Analytics jest prostszy i tańszy dla prostego filtrowania (CRITICAL eventy → SIEM). Databricks jest potrzebny dla złożonego enrichmentu (join z CMDB, deduplication CVE). Nie musisz wybierać — oba działają równolegle na tych samych Event Hubs.
PowerBI i Delta Lake: PowerBI Premium / Fabric obsługuje DirectQuery na Delta Lake przez Synapse lub Databricks SQL Warehouse. Bez Premium: Synapse Serverless SQL jako pośrednik (tańsze). Import mode (scheduled refresh) jest najszybszy dla dashboardów ale dane mają max 1h opóźnienie.
Koszt vs open source: Event Hubs ~$200–500/m (Standard tier, 20 TUs), Databricks ~$500–2000/m (zależy od DBU), Synapse Serverless ~$5/TB queries, ADLS ~$20/TB. Łącznie: ~$1000–3000/m za cały managed stack. Open source jest tańszy w licencjach, droższy w ludziach. Break-even przy braku dedykowanego data engineering teamu.
